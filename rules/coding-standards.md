# コーディング規約

このルールはコードファイルを編集する際にのみロードされる（path-scoped）。
プロジェクト固有の規約があれば、このファイルを編集するか、各言語ごとのルールファイルを `rules/` に追加してください。

## 命名規則

<!-- プロジェクトに合わせて書き換えてください -->
- 変数・関数: `camelCase`（JavaScript/TypeScript）または `snake_case`（Python/Rust）
- クラス・型: `PascalCase`
- 定数: `UPPER_SNAKE_CASE`
- ファイル名: `kebab-case` または言語慣習に従う

## コメント方針

- コードが「何をするか」ではなく「なぜそうするか」を説明する
- 自明な処理にはコメント不要
- 非自明な制約・回避策・外部仕様参照がある場合のみコメントを追加

## エラー処理

- エラーを握り潰さない（silent catch は禁止）
- ユーザー入力・外部 API の境界でのみバリデーション
- 内部ロジックのエラーは早期に失敗させる（fail-fast）

## テスト

- テストは実装と同じディレクトリ構造で管理
- ハッピーパスとエッジケース（null、空、境界値）を必ずカバー
- モックは最小限。DB 等の外部依存は統合テストで実環境を使う
