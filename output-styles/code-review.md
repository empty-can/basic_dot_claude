# コードレビュー出力スタイル

コードレビュー結果は以下のフォーマットで出力してください。

## フォーマット

### ヘッダー

```
## コードレビュー結果

対象: <ファイル名 or PR名>
レビュー日: <YYYY-MM-DD>
重大度別件数: CRITICAL: N / IMPORTANT: N / SUGGESTION: N / POSITIVE: N
```

### 指摘事項

各指摘は以下の形式で記述します：

```
- **[CRITICAL]** `file:line` — <問題の説明>
  - 問題: <何が悪いか>
  - 修正案: <どう直すか>

- **[IMPORTANT]** `file:line` — <問題の説明>
  - 問題: <何が悪いか>
  - 修正案: <どう直すか>

- **[SUGGESTION]** `file:line` — <改善提案>
  - 提案: <どう改善できるか>

- **[POSITIVE]** — <良かった点>
```

### 重大度の定義

| ラベル | 意味 | 対応 |
|---|---|---|
| **CRITICAL** | セキュリティ問題・破壊的変更・データ損失リスク | マージ前に必ず修正 |
| **IMPORTANT** | 品質・保守性・パフォーマンスの問題 | 修正を強く推奨 |
| **SUGGESTION** | 改善提案・スタイル・可読性 | 任意対応 |
| **POSITIVE** | 適切な実装・良い判断 | 記録として残す |

### フッター

```
---
総評: <全体的な評価を2〜3文で>
```

## 記述ルール

- 指摘には必ず具体的なファイルパスと行番号を含める
- 問題の指摘のみは不可。必ず修正案を提示する
- 変更されていないコードへのコメントは最小限にする
- POSITIVE は少なくとも1件記載する（ゼロは不可）
