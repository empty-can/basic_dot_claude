## 引数

`/review-skill-request [フォルダ名]` の形式で呼び出す。

- `フォルダ名` を省略した場合: `.claude/workspace/skill-request/` 配下のフォルダを列挙し、対象を確認する
- `フォルダ名` を指定した場合: そのフォルダを直接使用する

## 実行手順

### 1. 対象フォルダの特定

引数が省略された場合は `.claude/workspace/skill-request/` 配下のフォルダ一覧を取得し、
複数あればどれをレビューするか確認する。1つだけなら自動選択。

### 2. 依頼書を読み込む

対象フォルダの `skill-request-form.md` を読み込む。

読み込み後、依頼内容の複雑さを評価する:

- **複雑と判断する基準**: 高度なトレードオフ分析・広範な調査・複数の実現方式の比較検討が必要な場合
- **対応方針**:
  - 複雑な場合: `general-purpose` Agent（`model: opus`）を立て、手順 3〜5 を委任する
  - 委任が適切でない、またはAgentが利用できない場合: 作業を中断し、「この依頼内容は Opus での処理を推奨します。モデルを切り替えて `/review-skill-request` を再実行してください」と伝える
  - 複雑でない場合: そのまま手順 3 へ進む

### 3. 依頼書をレビューする

以下の観点でレビューし、各セクションの記載を評価する:

**完全性チェック**
- 未記入・「記入してください」のままのセクションはないか
- ユースケースが具体的か（「〜したい」だけでなく状況・入力・期待結果が書かれているか）
- 成功条件が検証可能か（曖昧な表現になっていないか）

**一貫性チェック**
- 機能要件とユースケースが矛盾していないか
- 実装制約が機能要件と矛盾していないか

**実現可能性チェック**
- Claude Code の Skill として実現可能な要件か
- 引数・呼び出し形式が Skill の仕様（スラッシュコマンド）と合っているか

### 4. 既存 Skill・Plugin を調査する

以下を順に確認する:

1. **本キット内**: `.claude/skills/` 配下に同等機能の Skill がないか
2. **導入済みプラグイン**: `/plugin` で導入済みプラグインに同等機能がないか。公式プラグイン集をローカルへ
   clone 済みなら、そのディレクトリも探す（**パスは各自の環境のもの**。作業指示者に所在を確認する）

類似機能が見つかった場合は、完全一致・部分一致・参考になる程度を区別して記録する。

### 5. skill-cc-response.md に書き込む

対象フォルダの `skill-cc-response.md` を以下の内容で更新する:

- **確認が必要な点**: 曖昧・未記入・情報不足の箇所を箇条書きで列挙。なければ「なし」
- **指摘事項**: 矛盾・実現困難・リスクがあれば記載。なければ「なし」
- **より良い方法の提案**: 要件を満たしつつよりシンプルな実現方法があれば提示。なければ「なし」
- **調査結果：既存 Skill・Plugin**: 調査結果を記載。見つからなければ「該当なし」
- **実装方針（合意前ドラフト）**: 「確認が必要な点」が残っている場合、その確認が必要な項目に依存する他のセクション（指摘・提案・実装方針）は無理に記載せず空欄のままにしてよい。「確認が必要な点」がなし、または影響しないセクションのみ先行して記載する。

### 6. 完了を報告する

レビュー結果に応じて以下のいずれかを出力する:

**確認事項がすべてクリアの場合**:
```
確認事項は全てクリアになりました。実装に進むことができます。
```
さらに、現在 Sonnet を使用しており、依頼内容が複雑または実装規模が大きいと判断される場合は以下を追記する:
```
なお、この内容は Opus での実装を推奨します。モデルを切り替えてから実装を依頼してください。
```

**確認事項が残っている場合**:
```
`skill-cc-response.md` の「確認が必要な点」への回答を `skill-request-form.md` に追記後、再度 `/review-skill-request` を実行してください。
```
このメッセージ出力後、以下のコマンドでシステムデフォルトのエディタで `skill-cc-response.md` を開く:
```
powershell.exe -Command "Start-Process '<skill-cc-response.md の絶対パス>'"
```
