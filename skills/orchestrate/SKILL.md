# オーケストレーション・フレームワーク

このスキルは、複数の Sub-agent を協調させて複雑なタスクを効率的に処理する手順を提供します。

Claude Code の公式仕様では「Subagents cannot spawn other subagents」のため、
本スキルは**メインセッション**で実行され、`Agent` ツール経由で子エージェントを並列/順次起動します。

## パターン A: 並列調査（Parallel Research）

複数の独立したトピックを同時に調査する場合に使用します。

**手順**:
1. タスクを独立したサブトピックに分割
2. 1 メッセージで複数の Agent ツール呼び出しを並列実行
3. 全エージェントの結果が揃った後に統合・分析

**例**: 「認証方式 A と B を同時に調査して比較」

## パターン B: 段階的処理（Sequential Chain）

前段の結果が次段の入力になる場合に使用します。

**手順**:
1. Phase 1: Explore エージェントでコードベースを調査
2. Phase 2: Plan エージェントで実装計画を策定（Phase 1 の結果を渡す）
3. Phase 3: code-reviewer エージェントで成果物をレビュー

**例**: 「既存コードを調査してからリファクタリング計画を立案」

## パターン C: 役割分担（Divide and Conquer）

大規模タスクを専門領域に分割して並列処理する場合に使用します。

**手順**:
1. タスクを専門領域（フロントエンド・バックエンド・テスト等）に分割
2. 各領域を専門エージェントに並列割り当て
3. メインセッションが全結果を取りまとめて統合

**例**: 「フロントエンドとバックエンドの変更を同時に実装」

## 実行のガイドライン

- **並列化できるものは並列化する**: 1 メッセージで複数 Agent ツール呼び出しを行う
- **コンテキストを明示して渡す**: 子エージェントはメインの会話コンテキストを持たないため、必要情報をプロンプトに含める
- **結果の統合はメインセッションで**: 子エージェントの出力を受け取り、最終的な判断と統合はメインセッションが担う
